== Safety Guarantees

Disk Ambulance follows *AmbientOps safety principles*:

=== âœ“ Evidence-First, Not Fearware

No "1,847 CRITICAL DISK ERRORS" scareware. Disk Ambulance shows you *measurements*, not hype.

Instead of: "Your disk is 99% failed!"

You get: "5 reallocated sectors detected (normal wear: <20, failing threshold: >100). Current status: MONITOR. Predicted failure: >6 months. Recommendation: schedule replacement during next maintenance window."

=== âœ“ Non-Destructive by Default

Diagnostics are *completely read-only*. No changes without your approval.

Repairs:

* Create automatic timestamped backups (filesystem metadata, partition tables, configs)
* Prefer read-only checks before repairs (fsck -n, then fsck -p if safe)
* **NEVER touch user data without explicit consent**
* Provide undo receipts for all changes

=== âœ“ Explicit Approval Required

No silent changes. Every repair shows:

* What it will do (exact commands)
* Why it's needed (root cause)
* What can go wrong (risk assessment)
* How to undo it (rollback procedure)
* Estimated downtime (if any)

You approve each change (or use `--yes` if you're confident).

=== âœ“ Never Destroys Data

Disk Ambulance will **NEVER**:

* Format drives without triple-confirmation
* Delete user files without explicit consent
* Run destructive fsck without read-only check first
* Execute `dd`, `shred`, or data destruction commands
* Make partition table changes without backup
* Force-mount corrupted filesystems (leads to data loss)

=== âœ“ Works Offline

Disk Ambulance doesn't require internet to diagnose or repair disk problems.

All diagnostics and repairs work *completely offline*.

=== âœ“ SMART-Aware Safety

Disk Ambulance checks SMART status *before* any repair:

* If drive is failing â†’ refuse repairs, suggest data migration
* If pending sectors exist â†’ force reallocation first, then repair filesystem
* If temperature critical â†’ wait for cooling, don't stress drive further

=== Requirements

* D compiler (DMD or LDC) for orchestration
* Rust compiler for safety-critical repairs
* Standard Linux utilities: `smartctl`, `fsck`, `df`, `lsblk`
* Optional: `mdadm` (RAID), `btrfs` (Btrfs), `xfs_repair` (XFS)
* Root/sudo for repairs (diagnostics work without root for some checks)

=== Implementation Language

Disk Ambulance uses a *hybrid implementation*:

* **D** for core orchestration (robust, cross-distro, good error handling)
* **Rust** for safety-critical filesystem operations (memory safety, no data corruption)
* **Bash** for simple wrappers and system integration

This follows Rhodium Standard Repository (RSR) guidelines.

=== Safety-Critical Operations (Rust)

The following operations are implemented in Rust for maximum safety:

* Filesystem metadata backup/restore
* Partition table reading/writing
* RAID metadata operations
* Direct device I/O (when needed)
* Filesystem repair orchestration (calls fsck but validates results)
