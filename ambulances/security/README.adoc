=== ğŸ”§ Safe, Auditable Hardening

When Security Ambulance finds vulnerabilities, it can fix themâ€”*safely*:

* **Firewall hardening**: Close unnecessary ports, add rate limiting, enable default deny, block known bad IPs
* **Service hardening**: Disable unnecessary services, enforce principle of least privilege, restrict service capabilities
* **Permission fixes**: Fix world-writable files, secure SUID binaries, harden SSH keys, protect config files
* **Access control**: Enable SELinux/AppArmor, configure policies, enforce mandatory access control
* **SSH hardening**: Disable password auth, enforce key-based authentication, disable root login, change default port
* **Update management**: Install security patches, configure auto-updates, verify package signatures

Every hardening action:

== Safety Guarantees

Security Ambulance follows *AmbientOps safety principles*:

=== âœ“ Evidence-First, Not Fearware

No "9,847 CRITICAL THREATS DETECTED" scareware. Security Ambulance shows you *actual risks*, not hype.

Instead of: "Your system is 99% vulnerable! Buy premium now!"

You get: "Port 3306 (MySQL) exposed to 0.0.0.0 with no password. CIS Benchmark 3.5.2 violation. Recommendation: restrict to localhost."

=== âœ“ Non-Destructive by Default

Security audits are *completely read-only*. No changes without your approval.

Hardening actions:

* Create automatic timestamped backups of all modified files
* Prefer additive security (add rules, don't delete config)
* Never lock you out (verify SSH keys before disabling passwords)
* Provide undo receipts for all changes

=== âœ“ Explicit Approval Required

No silent changes. Every hardening action shows:

* What it will do
* Why it's a security risk
* What compliance standards it violates
* How to undo it
* What could break

You approve each change (or use `--yes` if you're confident).

=== âœ“ Never Opens Security Holes

Security Ambulance will **NEVER**:

* Weaken existing security controls
* Disable firewalls or access controls
* Create new attack surfaces
* Expose services to public internet without explicit consent
* Disable SELinux/AppArmor without strong justification
* Store passwords or credentials
* Reduce password complexity requirements
* Open backdoors or "remote assistance" ports

=== âœ“ Compliance-Aware

Security Ambulance understands industry standards:

* CIS Benchmarks (Center for Internet Security)
* NIST Cybersecurity Framework
* DISA STIG (Security Technical Implementation Guides)
* PCI-DSS (Payment Card Industry)
* HIPAA Security Rule

All recommendations cite specific compliance requirements.

=== âœ“ Works Offline

Security Ambulance doesn't require internet to audit or harden systems.

All security checks and most hardening actions work *completely offline*.
