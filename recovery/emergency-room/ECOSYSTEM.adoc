== Interaction Flows

=== Emergency Flow
[source]
----
User notices problem
        │
        ▼
┌───────────────┐
│ emergency-    │ ──────────────────────────────────┐
│ button        │                                   │
└───────┬───────┘                                   │
        │                                           │
        ├── Disk full? ──▶ Clear caches, report     │
        │                                           │
        ├── Memory? ────▶ Kill safe processes       │
        │                                           │
        ├── Network? ───▶ complete-internet-repair ─┘
        │
        ▼
┌───────────────┐
│ Problem       │
│ resolved?     │
└───────┬───────┘
        │
   Yes  │  No
        │   │
        │   └──▶ big-up (deep diagnostics)
        │              │
        ▼              ▼
┌───────────────┐ ┌───────────────┐
│ feedback-     │ │ personal-     │
│ o-tron        │ │ sysadmin      │
│ (report)      │ │ (ongoing)     │
└───────────────┘ └───────────────┘
----

[source]
----

== Design Principles

1. **Immediate tools need no config** — emergency-button works out of the box
2. **Progressive complexity** — simple tools for simple problems, power tools available
3. **Offline-first** — crisis tools must work without network
4. **Safe defaults** — never make things worse
5. **Handoff protocols** — tools know when to defer to specialists
6. **Unified reporting** — all tools can feed into feedback-o-tron
